# 新服务器部署指南（hz.shandongliandong.com / 103.163.47.83）

将后端从旧服务器（144.123.115.42，juxinsuanli.cn）迁移到新服务器，
供全球云智算 App、小程序与管理后台使用。

前提：`hz.shandongliandong.com` 已解析到 `103.163.47.83`（✅ 已完成）。

---

## 一、新服务器初始化

```bash
# 安装 Docker、Nginx、certbot（以 Ubuntu/Debian 为例；CentOS 用 yum/dnf）
apt update
apt install -y docker.io docker-compose-v2 nginx certbot
systemctl enable --now docker nginx

# 项目目录（与旧服务器保持一致，nginx 配置里的路径都基于它）
mkdir -p /www/wwwroot/LD-AI
```

## 二、上传项目代码

代码托管在 GitHub 私有仓库（MS-219 账号下）。服务器上用**部署密钥（Deploy Key）**
拉取，只读权限、不暴露个人账号：

```bash
# ―― 在新服务器上执行 ――

# 1. 生成服务器专用的 SSH 密钥（一路回车，不要设密码）
ssh-keygen -t ed25519 -C "deploy@hz-server" -f ~/.ssh/id_ed25519

# 2. 打印公钥，复制输出内容
cat ~/.ssh/id_ed25519.pub
```

把公钥添加到 GitHub 仓库：仓库页面 → **Settings → Deploy keys → Add deploy key**，
标题随意（如 `hz-server`），粘贴公钥，**不勾选** Allow write access，保存。

```bash
# 3. 克隆到部署目录（仓库名以实际为准）
git clone git@github.com:MS-219/<仓库名>.git /www/wwwroot/LD-AI
cd /www/wwwroot/LD-AI
```

以后每次更新代码只需：

```bash
cd /www/wwwroot/LD-AI
git pull
./deploy-docker.sh deploy    # 重新构建并重启后端 + 管理后台
```

## 三、迁移 MySQL 数据

旧服务器上的 MySQL 跑在 `127.0.0.1:3307`，库名 `juxinsuanli`。

```bash
# 1. 旧服务器上导出（先摸清 mysql 容器名：docker ps | grep -i mysql）
mysqldump -h127.0.0.1 -P3307 -ujuxinsuanli -pjuxinsuanli \
  --single-transaction --routines juxinsuanli > /tmp/juxinsuanli.sql

# 2. 传到新服务器
scp /tmp/juxinsuanli.sql root@103.163.47.83:/tmp/

# 3. 新服务器上启动 MySQL 容器（端口/账号与旧环境保持一致，后端配置零改动）
docker run -d --name mysql \
  --restart unless-stopped \
  -p 127.0.0.1:3307:3306 \
  -e MYSQL_ROOT_PASSWORD='请改成强密码' \
  -e MYSQL_DATABASE=juxinsuanli \
  -e MYSQL_USER=juxinsuanli \
  -e MYSQL_PASSWORD=juxinsuanli \
  -v /www/wwwroot/mysql-data:/var/lib/mysql \
  mysql:8 --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci

# 4. 导入数据
mysql -h127.0.0.1 -P3307 -ujuxinsuanli -pjuxinsuanli juxinsuanli < /tmp/juxinsuanli.sql
```

> 上传文件（头像/收款码等）也要迁：
> `rsync -avz root@旧服务器:/www/wwwroot/LD-AI/backend/uploads/ root@103.163.47.83:/www/wwwroot/LD-AI/backend/uploads/`

## 五、签发 HTTPS 证书并配置 Nginx

```bash
# 1. certbot webroot 目录
mkdir -p /var/www/certbot

# 2. 先用仅 80 端口的临时配置让 certbot 能通过验证
cp /www/wwwroot/LD-AI/deploy/nginx-hz.conf /etc/nginx/conf.d/hz.conf
# 首次签发前证书文件还不存在，443 的 server 块会让 nginx 报错，
# 先注释掉 443 整个 server 块（或临时只保留 80 块），然后：
nginx -t && systemctl reload nginx

# 3. 签发证书（Let's Encrypt 免费，自动续期）
certbot certonly --webroot -w /var/www/certbot -d hz.shandongliandong.com

# 4. 恢复完整配置（取消 443 注释），重载
nginx -t && systemctl reload nginx

# 5. 自动续期已由 certbot 的 systemd timer 处理，可验证：
certbot renew --dry-run
```

## 七、收尾

- 设备端（device_agent）如果写死了旧域名，需要同步下发新配置/升级包

  并在微信公众平台「开发设置 → 服务器域名」添加 `https://hz.shandongliandong.com`
- 都验证无误后，旧服务器保留观察一段时间再下线；
  旧域名的图片 URL 已在 App 端做了自动替换，不影响历史数据显示
