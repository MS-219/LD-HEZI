# 新服务器部署指南（hz.shandongliandong.com / 103.163.47.83）

将后端从旧服务器（144.123.115.42，juxinsuanli.cn）迁移到新服务器，
供全球云智算 App、小程序与管理后台使用。

前提：`hz.shandongliandong.com` 已解析到 `103.163.47.83`（✅ 已完成）。

---

## 一、新服务器初始化

```bash
# 安装 Docker、Nginx、certbot（以 Ubuntu/Debian 为例；CentOS 用 yum/dnf）
apt update
apt install -y docker.io docker-compose-v2 nginx certbot
systemctl enable --now docker nginx

# 项目目录（与旧服务器保持一致，nginx 配置里的路径都基于它）
mkdir -p /www/wwwroot/LD-HEZI
```

## 二、上传项目代码

代码托管在 GitHub 私有仓库（MS-219 账号下）。服务器上用**部署密钥（Deploy Key）**
拉取，只读权限、不暴露个人账号：

```bash
# ―― 在新服务器上执行 ――

# 1. 生成服务器专用的 SSH 密钥（一路回车，不要设密码）
ssh-keygen -t ed25519 -C "deploy@hz-server" -f ~/.ssh/id_ed25519

# 2. 打印公钥，复制输出内容
cat ~/.ssh/id_ed25519.pub
```

把公钥添加到 GitHub 仓库：仓库页面 → **Settings → Deploy keys → Add deploy key**，
标题随意（如 `hz-server`），粘贴公钥，**不勾选** Allow write access，保存。

```bash
# 3. 克隆到部署目录（仓库名以实际为准）
git clone git@github.com:MS-219/<仓库名>.git /www/wwwroot/LD-HEZI
cd /www/wwwroot/LD-HEZI
```

以后每次更新代码只需：

```bash
cd /www/wwwroot/LD-HEZI
git pull
./deploy-docker.sh deploy    # 重新构建并重启后端 + 管理后台
```

## 三、初始化 MySQL（全新平台：只迁表结构和配置，业务数据从零开始）

全球云智算是从旧平台（LD-AI / 聚芯算力）分离出的**独立新平台**，
用户、设备、订单、收益全部从零开始；但后端（MyBatis-Plus）不会自动建表，
表结构和配置类数据需要从旧库导出。

```bash
# ―― 1. 旧服务器（144.123.115.42）上导出 ――

# 1a. 全部表结构（不含数据）
mysqldump -h127.0.0.1 -P3307 -ujuxinsuanli -pjuxinsuanli \
  --no-data --routines juxinsuanli > /tmp/schema.sql

# 1b. 配置类数据（管理员账号、系统配置、兑换商品、设备升级包）
mysqldump -h127.0.0.1 -P3307 -ujuxinsuanli -pjuxinsuanli \
  --no-create-info juxinsuanli \
  sys_user system_config exchange_product device_upgrade_package \
  > /tmp/seed.sql

# 1c. 传到新服务器
scp /tmp/schema.sql /tmp/seed.sql root@103.163.47.83:/tmp/
```

```bash
# ―― 2. 新服务器上建库导入 ――

# 2a. 启动本项目专用 MySQL 容器（不与 depin/sichuan 的库混用）
docker run -d --name qqyzs-mysql \
  --restart unless-stopped \
  -p 127.0.0.1:3307:3306 \
  -e MYSQL_ROOT_PASSWORD='请改成强密码' \
  -e MYSQL_DATABASE=juxinsuanli \
  -e MYSQL_USER=juxinsuanli \
  -e MYSQL_PASSWORD=juxinsuanli \
  -v /www/wwwroot/qqyzs-mysql-data:/var/lib/mysql \
  mysql:8.0 --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci

# 等 30 秒初始化完成，再导入
sleep 30
docker exec -i qqyzs-mysql mysql -ujuxinsuanli -pjuxinsuanli juxinsuanli < /tmp/schema.sql
docker exec -i qqyzs-mysql mysql -ujuxinsuanli -pjuxinsuanli juxinsuanli < /tmp/seed.sql

# 验证：应列出 26 张表
docker exec qqyzs-mysql mysql -ujuxinsuanli -pjuxinsuanli juxinsuanli -e "SHOW TABLES;"
```

> - 库名/账号沿用 `juxinsuanli` 是为了后端配置零改动（对外不可见）。
> - 公告（notice）没带过来——新平台开业公告在管理后台发新的即可。
> - uploads 不用迁：业务数据从零开始，没有历史图片。
> - 兑换商品如需重新定价，导入后在管理后台改。

## 四、启动后端

> ⚠️ 该服务器 8080 端口已被其他 Docker 容器占用，后端改用 **8090**
>（nginx-hz.conf 的反代地址已同步指向 8090）。

```bash
cd /www/wwwroot/LD-HEZI
SERVER_PORT=8090 ./deploy-docker.sh deploy   # 构建后端镜像 + 两个管理后台前端，并启动容器

# 验证
curl http://127.0.0.1:8090/api/notice/list
```

> 以后每次 `git pull` 后重新部署，也要带上 `SERVER_PORT=8090`。
> 想省事可以写进环境：`echo 'export SERVER_PORT=8090' >> /root/.bashrc && source /root/.bashrc`，
> 之后直接 `./deploy-docker.sh deploy` 即可。

## 五、签发 HTTPS 证书并配置 Nginx

```bash
# 1. certbot webroot 目录
mkdir -p /var/www/certbot

# 2. 先用仅 80 端口的临时配置让 certbot 能通过验证
cp /www/wwwroot/LD-HEZI/deploy/nginx-hz.conf /etc/nginx/conf.d/hz.conf
# 首次签发前证书文件还不存在，443 的 server 块会让 nginx 报错，
# 先注释掉 443 整个 server 块（或临时只保留 80 块），然后：
nginx -t && systemctl reload nginx

# 3. 签发证书（Let's Encrypt 免费，自动续期）
certbot certonly --webroot -w /var/www/certbot -d hz.shandongliandong.com

# 4. 恢复完整配置（取消 443 注释），重载
nginx -t && systemctl reload nginx

# 5. 自动续期已由 certbot 的 systemd timer 处理，可验证：
certbot renew --dry-run
```
