# 全球云智算 - Windows 镜像封装指南

## 概述
本指南介绍如何将 Windows 系统与全球云智算 Device Agent 封装成可批量部署的系统镜像。

---

## 准备工作

### 所需文件
- `device_agent.exe` - 算力节点程序
- `install_agent.bat` - 安装脚本

### 所需工具（选一种）
- **方案A**：微PE工具箱 + Ghost（推荐新手）
- **方案B**：Windows ADK + Sysprep（专业方案）
- **方案C**：优启通/大白菜等第三方PE工具

---

## 方案A：使用 Ghost 封装（推荐）

### 第一步：准备母机
1. 安装一台干净的 Windows 10/11 系统
2. 完成初始设置，进入桌面
3. 安装必要的驱动程序（如果是通用镜像，可跳过）

### 第二步：安装 Device Agent
1. 将 `device_agent.exe` 和 `install_agent.bat` 复制到桌面
2. **右键** `install_agent.bat` → **以管理员身份运行**
3. 等待安装完成

### 第三步：清理系统（可选但推荐）
```batch
# 清理临时文件
del /q/f/s %TEMP%\*
del /q/f/s C:\Windows\Temp\*

# 清理更新缓存
Dism.exe /online /Cleanup-Image /StartComponentCleanup /ResetBase
```

### 第四步：制作 Ghost 镜像
1. 制作一个 PE 启动U盘（使用微PE/优启通等）
2. 用 PE U盘启动母机
3. 打开 Ghost 工具
4. 选择 `Local` → `Disk` → `To Image`
5. 选择系统盘（通常是第一个磁盘）
6. 保存为 `.gho` 文件到U盘或其他存储

### 第五步：部署到新设备
1. 用 PE U盘启动新设备
2. 打开 Ghost 工具
3. 选择 `Local` → `Disk` → `From Image`
4. 选择之前保存的 `.gho` 文件
5. 选择目标磁盘，确认恢复
6. 重启后自动进入算力界面

## 方案B：使用 Sysprep 封装（专业）

### 第一步：准备母机
同方案A

### 第二步：安装 Device Agent
同方案A

### 第三步：运行 Sysprep
1. 以管理员身份打开 CMD
2. 执行以下命令：
```batch
C:\Windows\System32\Sysprep\sysprep.exe /oobe /generalize /shutdown
```
3. 等待系统自动关机

### 第四步：导出 WIM 镜像
1. 用 PE 启动，挂载母机硬盘
2. 执行 DISM 命令：
```batch
dism /Capture-Image /ImageFile:D:\JuxinSuanli.wim /CaptureDir:C:\ /Name:"LD Computing Node v1.0"
```

### 第五步：部署到新设备
```batch
# 在新设备的 PE 环境中
diskpart
select disk 0
clean
create partition primary
format fs=ntfs quick
assign letter=C
exit

dism /Apply-Image /ImageFile:D:\JuxinSuanli.wim /Index:1 /ApplyDir:C:\
bcdboot C:\Windows
```

## 注意事项

### 关于设备 SN
- 每台设备的 SN 是从 **硬件** 读取的（BIOS/主板/CPU/MAC）
- 所有 SN 自动添加 **LD** 前缀
- 同一镜像刷入不同设备，SN 不会冲突

### 关于设备信息显示 (v1.1 新功能)
启动后，屏幕布局如下：
- **左侧**：Matrix 算力效果（动态日志）
- **右侧**：设备信息面板，包含：
  - 📌 **设备 SN**：唯一标识符（如 `CDABC123DEF`）
  - 🔑 **绑定码**：6位短码（如 `A1B2C3`），方便用户手动输入绑定
  - 🌐 **IP 地址**：当前设备的局域网 IP
  - 🟢 **在线状态**：实时心跳检测状态
  - 📱 **二维码**：用户扫码可直接绑定设备

**用户绑定流程**：
1. 用户在小程序中选择「扫码绑定」
2. 扫描屏幕右侧的二维码
3. 确认绑定即可

### 关于 Kiosk 模式
- 安装脚本默认启用 Kiosk 模式
- 开机后直接显示算力界面，无桌面
- 如需恢复桌面，在 PE 中修改注册表：
```batch
reg load HKLM\OFFLINE C:\Windows\System32\config\SOFTWARE
reg add "HKLM\OFFLINE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v Shell /t REG_SZ /d explorer.exe /f
reg unload HKLM\OFFLINE
```

### 推荐配置
- Windows 10 LTSC（精简版，体积小）
- 关闭 Windows Update
- 关闭休眠和屏保

## 常见问题

**Q: 刷入后开机黑屏？**
A: 可能是显卡驱动问题，建议使用通用驱动或在封装前安装目标设备的驱动。

**Q: 设备无法联网？**
A: 检查网卡驱动，或在 PE 中预装通用网卡驱动。

**Q: 如何更新 Agent？**
A: 将新的 `device_agent.exe` 覆盖到 `C:\LD-AI\device_agent.exe` 即可。

## 方案D：Linux 镜像制作 (Ubuntu/Debian)

Linux 方案更适合大规模部署，资源占用更低。

### 第一步：准备母机
1. 安装 Ubuntu Server 20.04/22.04 LTS (建议最小化安装)
2. 配置好网络 (DHCP)

### 第二步：编译与安装
1. **编译 Linux 版 Agent** (在开发机上执行):
   ```bash
   SET CGO_ENABLED=0
   SET GOOS=linux
   SET GOARCH=amd64
   go build -o device_agent_linux main.go
   ```
2. **上传文件到母机**:
   将 `device_agent_linux` 和 `install_agent.sh` 上传到母机。
3. **执行安装**:
   ```bash
   chmod +x install_agent.sh
   sudo ./install_agent.sh
   ```

### 第三步：清理系统 (关键)
为了确保克隆后能生成新的唯一标识 (Machine ID, SSH Keys 等)，必须清理母机：

```bash
# 1. 清理 Machine ID (Systemd 生成新的)
sudo truncate -s 0 /etc/machine-id
sudo rm /var/lib/dbus/machine-id
sudo ln -s /etc/machine-id /var/lib/dbus/machine-id

# 2. 清理 SSH Host Keys (开机自动重新生成)
sudo rm -f /etc/ssh/ssh_host_*
# (可选) 添加 rc.local 脚本在首次启动时重新生成 keys
# 大多数现代发行版会自动处理这个

# 3. 清理网络配置 (如 Netplan/NetworkManager 的 UUID)
sudo rm -f /etc/netplan/*.yaml
# 重新创建一个通用的 DHCP 配置
echo "network: {version: 2, renderer: networkd, ethernets: {eth0: {dhcp4: true}}}" | sudo tee /etc/netplan/01-netcfg.yaml

# 4. 清理日志和临时文件
sudo rm -rf /var/log/*.log
sudo rm -rf /var/log/journal/*
sudo rm -rf /tmp/*
sudo history -c
```

### 第四步：制作镜像 (dd 或 Clonezilla)

#### 方法1：使用 Clonezilla (再生龙) - 推荐
1. 制作 Clonezilla 启动盘。
2. 启动母机，选择 "device-image" -> "savedisk"。
3. 生成镜像文件。
4. 批量刷机时使用 Clonezilla 的 "restoredisk" 模式。

#### 方法2：简单的 dd 命令 (适合相同硬盘大小)
```bash
# 在 PE 环境下
dd if=/dev/sda of=/path/to/backup.img bs=4M status=progress
```
